## 1. Что такое PR-TOP

**PR-TOP** — это AI-powered between-session ассистент для практикующих психологов и психотерапевтов. Платформа объединяет три компонента в единую экосистему:

1. **Веб-приложение** для терапевта — дашборд, аналитика, управление клиентами, AI-помощник
2. **Telegram-бот** для клиентов — дневники, упражнения, SOS, видео/голосовые сообщения
3. **AI-движок** — транскрипция, суммаризация сессий, семантический поиск, естественно-языковые запросы

**Главная идея:** дать терапевту полный контекст о клиенте между сессиями, не заставляя его вести двойную документацию.

## 2. Целевая аудитория

### 2.1 Основная ЦА

**Практикующие психологи и психотерапевты**, которые:
- Ведут от 5 до 50 клиентов одновременно
- Работают офлайн, онлайн или в гибридном формате
- Хотят повысить качество терапии, не увеличивая время на админку
- Готовы платить за инструменты, которые освобождают время и улучшают исходы

**География:** Россия, Украина, СНГ, ЕС, Латинская Америка (4 языка интерфейса: RU, EN, ES, UK).

### 2.2 Вторичная ЦА

- **Психологические клиники и центры** — для масштабирования работы команды
- **Супервизоры** — для контроля качества работы стажёров (через superadmin-роль)
- **Корпоративные психологи** — для работы с большими потоками клиентов

### 2.3 Профиль идеального клиента (ICP)

- Возраст: 28–55 лет
- Опыт: 3+ года практики
- Технологическая зрелость: средняя–высокая (пользуется онлайн-инструментами)
- Готовность к подписочной модели: да
- Болезненные точки: память, контекст, документация, безопасность данных

## 3. Боли клиента (Pain Points)

### 3.1 Боли терапевта

| Боль | Как сейчас решают | Почему это плохо |
|---|---|---|
| **Забывают, о чём говорили в прошлый раз** | Бумажные заметки, Excel, Notion | Время на чтение, риск пропустить важное |
| **Двойная документация** | Одна заметка вручную после сессии | Тратят 15–20 минут на каждую сессию |
| **Не знают, что происходит между сессиями** | Спрашивают на следующей встрече | Теряют 5–10 минут сессии на восстановление контекста |
| **Не помнят, какие упражнения давали** | Записывают в блокнот | Дублируют, забывают проверить выполнение |
| **Не успевают читать длинные дневники клиента** | Открывают перед сессией | Не успевают, читают по диагонали |
| **Не знают про кризис в реальном времени** | Узнают только на сессии | Упускают окно вмешательства |
| **Боятся за безопасность данных клиента** | Хранят в Google Docs / WhatsApp | Юридические риски, репутационные потери |
| **Не могут быстро найти информацию по клиенту** | Скроллят чаты и заметки | "Где-то она говорила про маму…" — поиск занимает минуты |
| **Транскрибируют сессии вручную** | Не делают вообще | Теряют ценные инсайты в потоке речи |
| **Тратят время на отчёты для супервизора** | Пишут вручную | 30–60 минут на отчёт |

### 3.2 Боли клиента (получателя терапии)

| Боль | Как PR-TOP помогает |
|---|---|
| Хочется поделиться чем-то важным между сессиями, но не хочется писать длинное сообщение | Голосовое сообщение в Telegram-бот → автоматическая транскрипция |
| Забывает, какое упражнение надо делать | Бот напоминает и принимает ответы |
| В кризисе нужна помощь, но не хочется звонить | Кнопка SOS — терапевт получает уведомление мгновенно |
| Боится, что его данные где-то "утекут" | Прозрачное согласие, шифрование, право отозвать доступ |
| Не хочет ставить ещё одно приложение | Использует Telegram, который уже стоит |

## 4. Как PR-TOP решает эти боли

### 4.1 Главные ценностные предложения (Value Propositions)

#### 🧠 **"Помни всё о каждом клиенте — без усилий"**
AI автоматически собирает контекст из дневников, сессий, заметок и упражнений в единый Timeline. Открыл клиента — увидел всё за секунду.

#### ⏱ **"Минус 15 минут на каждую сессию"**
Загрузил аудио сессии → AI транскрибирует и резюмирует. Никаких ручных заметок. Только дополнения, если нужно.

#### 🔒 **"Данные клиентов защищены лучше, чем в банке"**
AES-шифрование на уровне приложения. Класс А (дневники, расшифровки, заметки) шифруются перед записью в БД. Self-hosted, GDPR-compliant.

#### 🚨 **"Узнавай о кризисе, а не на следующей сессии"**
Кнопка SOS → push, email, Telegram-уведомление. Можно настроить escalation rules и quiet hours.

#### 💬 **"Спрашивай AI-ассистента про клиента естественным языком"**
"Когда клиент впервые упоминал маму?" → AI находит все записи с релевантным контекстом. Векторный семантический поиск.

#### 📱 **"Клиенту не надо ничего ставить"**
Работает в Telegram, который у клиента уже есть. Голос, видео, текст — всё в одном чате.

## 6. Безопасность и конфиденциальность

> Это **главное** конкурентное преимущество. Психологи в первую очередь продаются на безопасности.

### 6.1 Классификация данных

**Класс А — шифруется на уровне приложения (AES) перед записью в БД:**
- Содержимое дневников клиента (текст, голосовые транскрипты, видео транскрипты)
- Сообщения переписки
- Терапевтические заметки
- Саммари сессий
- Анамнез и контекст клиента
- AI-инструкции и контрпоказания
- Цитаты из SOS-событий
- Ответы клиентов на упражнения

**Класс B — access-controlled metadata (plaintext):**
- Timestamps
- ID связей терапевт-клиент
- Статусы и роли
- Языковые теги
- Платёжная metadata (Stripe сам обрабатывает PCI)
- UTM attribution

### 6.2 Меры защиты

| Мера | Реализация |
|---|---|
| **Аутентификация** | JWT с HttpOnly cookies (нельзя украсть через XSS) |
| **CSRF** | Double-submit cookie pattern |
| **Шифрование at rest** | AES для всех данных Класса А |
| **Key rotation** | Версионирование ключей с поддержкой re-encryption |
| **Пароли** | bcrypt с проверкой стойкости |
| **Rate limiting** | На auth endpoints (защита от brute force) |
| **Audit logging** | Все обращения к чувствительным данным логируются (actor, action, timestamp) |
| **Role-based access** | Route guards + API middleware (therapist/admin/client) |
| **Consent enforcement** | Все client-data маршруты проверяют consent_therapist_access |
| **File security** | Opaque IDs для файлов, signed-access streaming |
| **No data leakage** | Файлы сессий в `/data/sessions/` (вне веб-корня) |
| **Encrypted backups** | Backup snapshots зашифрованы master key |
| **HTTPS only** | Let's Encrypt + HSTS |
| **Security headers** | X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy |
| **GDPR compliance** | Право на экспорт, право на удаление, явное согласие |

### 6.3 Что мы НЕ делаем (это тоже маркетинг)

- ❌ **Не отправляем данные в Google/Facebook/etc** — Umami self-hosted, без cookies
- ❌ **Не используем cloud-trackers** — никаких Hotjar, Mixpanel, etc
- ❌ **Не храним пароли в открытом виде** — bcrypt
- ❌ **Не передаём данные клиента третьим лицам** — кроме самого терапевта
- ❌ **Не используем "always-on" telemetry** — только Umami на landing page
