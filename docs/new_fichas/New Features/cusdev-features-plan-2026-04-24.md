## Сводка ключевых инсайтов из интервью

| Источник | Главное |
|----------|---------|
| Миша (коуч/психолог) | Нужна иерархия **Запрос → Встреча → Задание → Упражнение**; отчёты по заданиям с приёмкой; двойная система комментариев (public/private); библиотека упражнений выглядит «детсадовской», без пометки «пример» |
| Алексей (психоаналитик) | Клиент в бот не придёт (паранойя, этика); нужен **соло-режим** «умной тетрадки»; upload аудио через админку не работает (**баг**); нужен кастомный summary под модальность (психоанализ ≠ КПТ ≠ НЛП); напоминалки не нужны |
| Общее | Запись сессии идёт во внешних инструментах (Zoom/Meet) → хочется интеграции; база знаний терапевта для AI; загрузка учебников/литературы своей школы |

## Предлагаемые фичи

### 🔴 Приоритет 1 — Критичные (меняют ядро продукта)

| # | Категория | Название | Описание | Из интервью |
|---|-----------|----------|----------|-------------|
| 1 | Hierarchy | **Сущность «Запрос» (Inquiry/Request)** | Над Сессией надстраивается сущность «Запрос» клиента (напр. «меньше срываться на близких»). Один клиент может иметь несколько параллельных/последовательных запросов, каждый содержит набор сессий. У запроса: название, дата открытия/закрытия, статус, описание цели. | Миша, стр.217–274 |
| 2 | Hierarchy | **Сущность «Встреча/Сессия» с датой-как-идентификатором** | Сессии ищутся/называются по дате (не по теме). Стандартный календарь для навигации. Внутри встречи: запись, транскрипт, summary, задания, комментарии. Опциональный заголовок-комментарий. | Миша, стр.46–86 |
| 3 | Hierarchy | **Задания (Assignments) привязанные к сессии** | В рамках встречи терапевт создаёт 1+ задание. Задание может включать упражнение из библиотеки ИЛИ быть свободным текстом под конкретного клиента. Поля: название, описание, частота отчётов (ежедневно/раз в N дней), дедлайн. | Миша, стр.110–181 |
| 4 | Reporting | **Свободные отчёты клиента по заданию** | Клиент пишет текст (не кликает чекбоксы) по ходу выполнения задания — сколько угодно раз. Возможность прикрепить фото. Отдельная лента внутри задания. | Миша, стр.125–196 |
| 5 | Reporting | **Итоговый отчёт + приёмка терапевта** | Когда клиент считает задание завершённым — оформляет итоговый отчёт. Терапевт либо «принимает», либо «возвращает на доработку» **с обязательным комментарием** — почему. Без комментария вернуть нельзя. | Миша, стр.303–345 |
| 6 | Personal Mode | **Соло-режим «Умная тетрадка» (therapist-only)** | Режим, где терапевт создаёт «клиента» вручную в админке (без Telegram-бота, без invite-кода). Клиент не знает о системе. Терапевт: записывает только свой голос/заметки, получает транскрипт+summary. Никаких дневников/упражнений со стороны клиента. Нужен для психоаналитиков и тревожных клиентов. | Алексей, стр.801–935 |
| 7 | Bug | **Загрузка аудио через админку (ныне отсутствует)** | На момент интервью Алексей не смог найти, куда загрузить аудио файл через веб-интерфейс — грузит только из Telegram. Нужна полноценная кнопка «Загрузить запись» внутри сессии клиента в дашборде. | Алексей, стр.94–101 |

### 🟠 Приоритет 2 — Высокие (меняют UX и качество AI)

| # | Категория | Название | Описание | Из интервью |
|---|-----------|----------|----------|-------------|
| 8 | AI | **Кастомные промпты summary под модальность** | Терапевт настраивает, что выделять в summary: психоаналитические маркеры, поведенческие паттерны (КПТ), якоря/субмодальности (НЛП) и т.д. Либо по шаблонам, либо свободный промпт. Поле «моя специализация» + опциональный системный промпт. | Алексей, стр.227–269 |
| 9 | AI | **Персональная база знаний терапевта для AI** | Терапевт загружает учебники, статьи, ссылки, литературу своей школы (разные языки). AI использует как RAG-источник при генерации summary и ответов на NL-запросы. | Алексей, стр.275–356 |
| 10 | Comments | **Двойная система комментариев к любой сущности** | К сессии/заданию/упражнению можно оставить **два** типа комментариев: (a) для себя — приватно, (b) видимый клиенту. Два отдельных поля. | Миша, стр.374–396 |
| 11 | Comments | **Видимость комментариев терапевта — по умолчанию скрыта** | Комментарий терапевта не виден клиенту, если терапевт явно не включил тумблер «показать». | Миша, стр.294–302 |
| 12 | Comments | **Приватные комментарии клиента** | Клиент может оставить личный комментарий (только для себя, терапевт не видит). По умолчанию комменты клиента видны терапевту, но есть тумблер «скрыть». | Миша, стр.285–297 |
| 13 | Exercise Library | **Пометка «Пример / Template» на всех библиотечных упражнениях** | Каждая карточка начинается со штампа «Пример оформления» (не «упражнение от PR-TOP»). Психологи отсекают сервис, увидев слабые формулировки — важно снять ответственность за содержание. | Миша, стр.410–484 |
| 14 | Exercise Library | **Чистка и профессионализация библиотеки упражнений** | Удалить/переписать слабые карточки (напр. «Рефрейминг» в текущем виде — слишком обобщённо и «ерунда не выдерживает»). Оставить конкретные: «Отслеживание привычек» — эталон. Передать проверку эксперту соответствующей школы. | Миша, стр.429–533 |
| 15 | Session Entity | **Поле «пост-сессионные заметки терапевта»** (голос+текст) | Отдельное поле внутри сессии: терапевт после окончания Zoom быстро наговаривает/пишет «на что обратить внимание в следующий раз». AI учитывает его в summary. | Алексей, стр.73–92 |

| 20 | Recording | **Авто-привязка аудио к сессии по дате/метаданным** | Если терапевт загружает несколько записей за день — система разбирает metadata файлов и подтягивает к нужному клиенту/слоту. Нужна логика конфликтов (несколько клиентов в один интервал). | Алексей, стр.195–213 |
| 21 | UX | **Attachments (фото) к отчётам клиента** | Клиент через бот прикладывает фото к отчёту по заданию (напр. «сфотографировал дневник», «записал тренировку»). | Миша, стр.51–53 |
| 22 | Exercise | **Комментарии клиента к каждому выполнению упражнения** | Во время выполнения стандартного упражнения из библиотеки — лента комментариев клиента. Итоговый отчёт после завершения. | Миша, стр.125–145 |

## Рекомендованный порядок реализации (Top-5 для ближайшего MVP)

1. **Иерархия Запрос → Сессия → Задание → Отчёт** (#1–5) — фундамент, без которого остальное не встаёт. Обе модели терапии это просили (Миша прямо как коуч-формат, Алексей косвенно через «жалко, что приходится вспоминать»).
2. **Соло-режим (#6)** — открывает сегмент психоаналитиков, который сейчас отсечён.
3. **Загрузка аудио через админку (#7)** — баг, простая фикс, разблокирует Алексея как тестового пользователя.
4. **Кастомный summary + база знаний (#8, #9)** — главный дифференциатор vs. generic-AI, запрос был у обоих.
5. **Пометка «Пример» на библиотеке (#13)** — минимальная работа, большой эффект на доверие со стороны психологов.

## Открытые вопросы для продолжения cusdev

- Нужна ли связь «Запрос ↔ набор шаблонов заданий» (коуч-вид)?
- Как обрабатывать кейс «клиент вернулся через год» — новое соединение в боте или переиспользование старого?
- Какой формат базы знаний удобнее терапевту: Google Drive, прямой upload, ссылки?
- Готовы ли психоаналитики платить за соло-режим, или это должно быть дешевле базового тарифа?
- Проверить: у скольких ещё потенциальных клиентов есть проблема с доступом из РФ?
